HTMX Triggers

Is it possible to have multiple trigger types in HTMX? Yes, the hx-trigger attribute allows multiple triggers. You can declare multiple triggers by separating them with commas. In the example below, you can see that the select field has two triggers - load and change. When the first trigger occurs on page load, the HTMX code will post the pre-selected option to the URL specified with hx-post (/api/search). The response from that server will then display on the page in div#result, due to the use of hx-target, which identifies the area to place the response, and hx-swap, which places it within the div. The second trigger, change, will be used to update the response when the user selects another city option from the select menu. 


<select name="city" id="city" hx-trigger="load, change" hx-post="/api/search" hx-target="#result" hx-swap="innerHTML">
    <option value="San Francisco">San Francisco</option>
    <option value="New York">New York</option>
    <option value="Las Vegas">Las Vegas</option>
</select>
<div id="result"></div>
